🚀 Spring Boot 4

Novedades Importantes y Guía de Migración

🎯 El Contexto

Spring Boot 4 fue lanzado oficialmente en Noviembre 2025.
Este curso usa Spring Boot 4, aunque muchos recursos en internet aún hablan de Spring Boot 3.

💡 Importante:

Spring Boot 3 NO está mal. Pero Boot 4 trae cambios importantes en rendimiento y arquitectura que son especialmente útiles para microservicios.

🧠 La Filosofía del Cambio

Spring Boot 3

"Opinado y Mágico"
  • Nosotros decidimos por ti
  • starter-web incluye TODO
  • Más fácil de empezar
  • Menos decisiones que tomar

Spring Boot 4

"Explícito y Optimizado"
  • Tú decides, nosotros optimizamos
  • Debes elegir: MVC o Reactive
  • Más control sobre dependencias
  • Tiempo de arranque más rápido
1️⃣
Starters Explícitos

El cambio más visible: ahora debes elegir explícitamente entre MVC o Reactive.

❌ Spring Boot 3
<artifactId>spring-boot-starter-web</artifactId>
✅ Spring Boot 4 (MVC Tradicional)
<artifactId>spring-boot-starter-webmvc</artifactId>
2️⃣
Java 21 y Virtual Threads

Spring Boot 4 recomienda Java 21 para usar Virtual Threads.

⚙️ Activación (application.yml)
spring:
threads:
virtual:
enabled: true
💻 Ejemplo Práctico
@GetMapping("/{id}")
public ProductResponse getProduct(@PathVariable String id) {
// Llamada HTTP a servicio externo (2 segundos)
Product product = productService.findById(id);
String imageUrl = externalService.getImage(product.getId());
 
return new ProductResponse(product, imageUrl);
}
🎯 El Resultado:

Sin Virtual Threads: Thread bloqueado 2 segundos → Límite 5,000 requests

Con Virtual Threads: Thread se "aparca" mientras espera → >1,000,000 requests

Sin escribir código reactivo complejo. Solo configuración.

3️⃣
Spring Cloud 2023.x - Compatibility Break
⚠️ IMPORTANTE: Incompatibilidad

Spring Boot 4 NO es compatible con Spring Cloud 2022.x (versión de Boot 3)
Debes usar Spring Cloud 2023.x o superior

❌ Esto NO funciona
<parent>
<artifactId>spring-boot-starter-parent</artifactId>
<version>4.0.1</version> <!-- Boot 4 -->
</parent>
 
<properties>
<spring-cloud.version>2022.0.4</spring-cloud.version> <!-- ❌ Boot 3 -->
</properties>
✅ La solución
<parent>
<artifactId>spring-boot-starter-parent</artifactId>
<version>4.0.1</version>
</parent>
 
<properties>
<spring-cloud.version>2023.0.0</spring-cloud.version> <!-- ✅ Boot 4 -->
</properties>
Componentes afectados:
  • Eureka Server/Client
  • Config Server
  • API Gateway
  • OpenFeign
  • Circuit Breaker

📊 Comparación Completa

Aspecto Spring Boot 3 Spring Boot 4
Java Java 17+ Java 21 recomendado
Web Starter starter-web (todo incluido) starter-webmvc o starter-webflux
Spring Cloud 2022.x (Kilburn) 2023.x+ (Leyton)
Tiempo Arranque ~2.8s ~1.9s (-32%)
Memoria (Idle) ~350 MB ~280 MB (-20%)

🤔 ¿Cuándo Migrar?

Árbol de Decisión

✅ Usa Spring Boot 4 si:

  • Estás empezando un proyecto nuevo
  • Quieres el mejor rendimiento
  • Usarás Java 21 (Virtual Threads)

⚠️ Mantente en Boot 3 si:

  • Proyecto en producción estable
  • Dependencias legacy no compatibles
  • Proyecto pequeño sin carga alta
